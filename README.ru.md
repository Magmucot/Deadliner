# Deadliner

Кроссплатформенная десктоп-утилита для напоминаний о перерывах и запланированных событиях, построенная на C++ и Qt 6.

[Readme in English](README.md)

## Возможности

- **Интеграция с системным треем** — работает в фоне, контекстное меню для быстрых действий
- **Интервальные напоминания о перерывах** — периодические уведомления с настраиваемыми интервалами и длительностью
- **Напоминания по дате/времени** — одноразовые и повторяющиеся события с настраиваемой повторяемостью (ежедневно, по будням, еженедельно, ежемесячно, каждые N дней)
- **Три режима напоминаний:**
  - **Мягкий** — пассивное уведомление, исчезает само
  - **Настойчивый** — рамочное перекрытие, остаётся видимым до действия пользователя
  - **Перерыв** — полноэкранный оверлей с обратным отсчётом, блокировкой пропуска и подтверждением возврата
- **Тихие часы** — поведение по режимам (подавить, отложить, понизить, разрешить), включая интервалы через полночь
- **Откладывание с лимитами** — ограничение количества откладываний на профиль и поддержка эскалации
- **Профили** — переиспользуемые профили напоминаний (перерыв, общий, смешанный) со встроенными и пользовательскими пресетами
- **Двуязычный интерфейс (RU/EN)** — переключение языка без перезапуска, определение языка системы, выбор языка при первом запуске
- **Темы** — светлая, тёмная и системная темы через стиль Fusion + QPalette
- **Мастер первого запуска** — настройка языка и рекомендованного правила перерывов при первом запуске
- **Отслеживание просроченных событий** — отдельная страница для пропущенных и просроченных напоминаний
- **Сводка на сегодня** — приветствие, обратный отсчёт до перерыва, активные профили и предстоящие события
- **Ежедневная статистика** — количество выполненных, пропущенных, отложенных и исходов перерывов
- **Хранение в SQLite** — профили, события, журналы и статистика сохраняются между запусками
- **Хранение в QSettings** — геометрия окна, тема, язык, поведение трея и другие настройки приложения
- **Автозапуск** — платформенные адаптеры для Windows (папка Startup) и Linux (`~/.config/autostart`)
- **CI/CD** — рабочий процесс GitHub Actions: NSIS-инсталлятор + портативный ZIP (Windows) и Arch-пакет + tarball (Linux)

## Быстрый старт

```bash
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build
./build/deadliner
```

## Требования для сборки

### Windows 10

- Visual Studio 2022 с MSVC
- CMake 3.24+
- Qt 6.5+ с модулями `Core`, `Gui`, `Widgets`, `Sql`, `Test`, `LinguistTools`

### Arch Linux / CachyOS

```bash
sudo pacman -S cmake ninja qt6-base qt6-tools sqlite
```

## Сборка

### Windows

```powershell
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build
ctest --test-dir build --output-on-failure
```

Развёртывание для портативного использования:

```powershell
windeployqt build\deadliner.exe
```

Для создания инсталлятора используйте развёрнутый вывод со скриптом NSIS: `packaging/windows/installer.nsi`.

### Linux

```bash
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release
cmake --build build
ctest --test-dir build --output-on-failure
```

Локальный запуск:

```bash
./build/deadliner
```

Установка в staging-префикс:

```bash
cmake --install build --prefix "$PWD/stage"
```

## Архитектура

Приложение следует практическому модульному монолиту с чётким разделением слоёв:

| Слой | Ответственность |
|---|---|
| **UI** | Qt Widgets: навигация через боковую панель, страницы, диалоги, экран перерыва, трей, онбординг |
| **Application** | `ReminderScheduler` (центральный QTimer + очередь с приоритетом), `StatisticsService` |
| **Domain** | Модели (`ReminderProfile`, `ReminderEvent`, `QuietHoursPolicy` и др.), логика планирования, правила тихих часов, политика откладывания |
| **Infrastructure** | `Repository` (SQLite + миграции), `SettingsStore` (QSettings), `AutostartManager` (платформенные адаптеры), логирование |

**Поток выполнения:**

1. Запуск приложения → загрузка настроек, профилей, событий, статистики из SQLite/QSettings
2. Инициализация трея, планировщика, автозапуска, языка, темы
3. Перестроение очереди событий в памяти из сохранённого состояния
4. Центральный `QTimer` срабатывает в момент наступления ближайшего события
5. Проверка тихих часов → применение политики профиля → отображение напоминания (мягкий / настойчивый / перерыв)
6. Действие пользователя (выполнено / отложено / пропущено) → обновление статистики → перестроение расписания

## Упаковка

### Windows

CI собирает NSIS-инсталлятор и портативный ZIP. Локально:

1. Собрать release-бинар.
2. Запустить `windeployqt` на `deadliner.exe`.
3. Запустить `makensis` со скриптом `packaging/windows/installer.nsi`.

### Arch Linux / CachyOS

1. Собрать через CMake.
2. Использовать `packaging/linux/PKGBUILD` с `makepkg` для нативного Arch-пакета.
3. Или установить в staging-директорию: `cmake --install build --prefix "$PWD/stage"`.

## Хранение данных

- **База данных SQLite:** `QStandardPaths::AppDataLocation` (профили, события, журналы, статистика)
- **Лог-файл:** `logs/deadliner.log` в той же директории данных приложения
- **QSettings:** платформенно-нативный бэкенд (настройки UI, тема, язык, поведение трея)

## Системные ограничения

- Поведение трея на Linux зависит от десктопного окружения и поддержки StatusNotifierItem/AppIndicator.
- Wayland может ограничивать надёжность удержания окна перерыва поверх других приложений.
- Приложение не использует инвазивные хуки и не может доказать, что пользователь физически отошёл от компьютера. Выполнение перерыва отслеживается по сценарию приложения (минимальная длительность + явное подтверждение), а не по физическому присутствию.
- Создание нативного инсталлятора и AppImage требует внешних инструментов упаковки, не встроенных в данный репозиторий.
