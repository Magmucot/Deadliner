# UI/UX Specification: Deadliner (Qt 6 Desktop App)

## 1. UI Goals
- Создать кроссплатформенное (Windows 10, Linux) desktop-приложение для управления перерывами и напоминаниями.
- Обеспечить минимальное трение при повседневном использовании: приложение не должно раздражать, когда это не нужно, но обязано добиваться внимания пользователя в строгих режимах.
- Обеспечить бесшовную двуязычность (RU/EN) "из коробки" без визуальных поломок интерфейса.
- Спроектировать интерфейс, который будет легко, предсказуемо и нативно реализовываться средствами **Qt 6 Widgets** (без переусложненных кастомных отрисовок).

## 2. Design Principles
1. **Unobtrusive but Escalating:** Напоминания начинаются мягко и усиливаются только если пользователь их игнорирует (или если так настроено правило).
2. **Robust Layouts:** Никаких фиксированных ширин. Тексты на русском языке длиннее английских в среднем на 20-30%, интерфейс должен "дышать" и переносить текст, а не обрезать его.
3. **Keyboard First & Accessibility:** Все основные окна (особенно диалоги напоминаний) должны закрываться, откладываться или подтверждаться с клавиатуры (`Enter`, `Esc`, `Space`, `Tab`).
4. **Desktop Native:** Использование парадигм десктопа (системный трей, контекстные меню, модальные окна, фоновая работа), а не перенос мобильного или веб-мышления.

## 3. Information Architecture
Система делится на 5 основных разделов:
1. **Dashboard (Сводка):** Статус текущего дня, таймер до следующего перерыва, ближайшие события.
2. **Break Rules (Правила перерывов):** Настройка циклов (например, 20-20-20, разминка каждый час).
3. **Events (События):** Одноразовые или повторяющиеся напоминания в конкретное время.
4. **Statistics (Статистика):** Графики соблюдения перерывов.
5. **Settings (Настройки):** Поведение приложения, автозапуск, выбор языка, тема (светлая/темная/системная).

## 4. Navigation Model
- **Тип навигации:** Вертикальная боковая панель (Sidebar) слева. Реализуется через стилизованный `QListWidget` или `QVBoxLayout` с `QPushButton` (checkable).
- **Контентная область:** Справа, переключается через `QStackedWidget`.
- **Быстрый доступ:** Иконка в системном трее (`QSystemTrayIcon`) с контекстным меню.

## 5. Screen Inventory
1. `MainWindow` (Главное окно, содержит навигацию и StackedWidget).
2. `DashboardView` (Главный экран).
3. `BreakRulesView` (Список правил перерывов).
4. `RuleEditorDialog` (Создание/редактирование правила).
5. `EventsView` (Список событий).
6. `EventEditorDialog` (Создание/редактирование события).
7. `StatisticsView` (Простая аналитика).
8. `SettingsView` (Глобальные настройки).
9. `ReminderSoftWidget` (Небольшой тост или использование OS Notifications).
10. `ReminderPersistentDialog` (Поверх всех окон, но не блокирует экран).
11. `ReminderStrictWindow` (Полноэкранный блокирующий оверлей).
12. `OnboardingDialog` (Приветствие и базовые настройки при первом запуске).

## 6. Core User Flows
- **Первый запуск (Onboarding):**
  Запуск -> Выбор языка (RU/EN) -> Краткое объяснение (3 слайда) -> Предложение включить стандартное правило "20-20-20" -> Готово (приложение уходит в трей/открывает Dashboard).
- **Включение режима перерывов:**
  Dashboard -> Перейти в Break Rules -> Нажать "Add Rule" -> Настроить интервал (например, 45 мин работа, 5 мин отдых) -> Выбрать строгость (Strict) -> Нажать Save.
- **Обязательный перерыв (Strict):**
  Время вышло -> Экран затемняется (открывается `ReminderStrictWindow` поверх всех окон) -> Таймер идет вниз -> Кнопка "Skip" недоступна первые 10 секунд -> По окончании таймера окно закрывается само.
- **Управление из трея:**
  Клик правой кнопкой по иконке -> Выбор "Pause reminders" -> Выбор "For 1 hour" -> Иконка в трее меняется на "Zzz" / серую.

## 7. Reminder Mode UX

| Режим | Поведение (UX) | Когда использовать | Откладывание (Snooze) |
| :--- | :--- | :--- | :--- |
| **Soft** (Мягкий) | Нативное уведомление ОС (через `QSystemTrayIcon::showMessage`). Само исчезает. | Напоминание выпить воды, начало неважного события. | Если поддерживается ОС (через действия уведомления) или игнор. |
| **Persistent** (Настойчивый) | Окно без рамки (`Qt::FramelessWindowHint`), `WindowStaysOnTopHint`. Появляется в углу или по центру, не перехватывает фокус клавиатуры грубо, но не исчезает само. | Напоминание о встрече, средняя усталость. | Легко доступна кнопка "Snooze" (5-10 мин). |
| **Strict** (Строгий) | Полноэкранный (`showFullScreen()`), полупрозрачный фон. Перехватывает фокус. | Большой перерыв, критическое событие (окончание рабочего дня). | Ограничено. Либо Snooze доступен только N раз, либо кнопка "Skip" заблокирована первые X секунд (Anti-abuse). |

**Escalation Logic:** Мягкое уведомление за 2 минуты до -> Если проигнорировано -> Строгий экран.

## 8. Localization UX (RU/EN)
- **Стратегия языка по умолчанию:** При первом запуске берется из системы (`QLocale::system().language()`). Если не русский — ставим английский.
- **Где меняется:** Во время Onboarding и в Настройках.
- **Live Language Switch:** **ДА.** Реализуется через переопределение метода `changeEvent(QEvent *event)` и вызов `ui->retranslateUi(this)` при событии `QEvent::LanguageChange`. Перезапуск не требуется, это дает эффект премиальности.
- **Рискованные зоны (Переполнение):** Кнопки действий (Skip/Отложить), заголовки таблиц, настройки.

## 9. Key Screen Specs

### 9.1 Dashboard (Сводка)
**Цель:** Дать понимание "что сейчас происходит и когда следующий перерыв".
**Примечания по локализации:** Блок "Next break" должен использовать компоновку, где текст и время не прибиты гвоздями, чтобы избежать наложений.

```text
+-------------------------------------------------------------+
|        |                                       [Quiet Mode] |
| [Icon] |  Good morning, User! (Доброе утро!)                |
| Dashbd |                                                    |
|        |  Next break in: (Следующий перерыв через:)         |
| Breaks |  [       14:32       ]                             |
|        |                                                    |
| Events |  Active Rules: (Активные правила:)                 |
|        |  (o) 20-20-20 Eye Rest       [Edit] [Toggle]       |
| Stats  |  ( ) Stand Up (Every 1h)     [Edit] [Toggle]       |
|        |                                                    |
| Settng |  Today's Events: (События на сегодня:)             |
|        |  - 15:00 Team Sync (Soft)                          |
+-------------------------------------------------------------+
```

### 9.2 Strict Break Mode (Строгий перерыв)
**Цель:** Заставить пользователя отдохнуть, минимизируя возможность моментального закрытия по привычке.
**UX:** Экран слегка затемняет фон (Opacity 80%, blur если возможно, иначе просто черный полупрозрачный цвет). Большой таймер. Кнопка "Пропустить" появляется не сразу или требует зажатия.

```text
+-------------------------------------------------------------+
|                                                             |
|                                                             |
|                   TIME TO REST YOUR EYES                    |
|                      (ВРЕМЯ ОТДОХНУТЬ)                      |
|                                                             |
|                            00:19                            |
|                                                             |
|                                                             |
|   [ Skip (Пропустить) ]               [ Snooze (Отложить) ] |
|   (Disabled for 10s)                                        |
|                                                             |
+-------------------------------------------------------------+
```

## 10. Tray UX
- Приложение живет в трее. Закрытие крестиком в главном окне сворачивает в трей (настраиваемо).
- **Контекстное меню (`QMenu`):**
  - Окно управления (Open Dashboard)
  - ---
  - Приостановить на 1 час (Pause for 1 hour)
  - Приостановить на 3 часа (Pause for 3 hours)
  - Приостановить до завтра (Pause until tomorrow)
  - ---
  - Настройки (Settings)
  - Выход (Quit)

## 11. Onboarding and Empty States
- **Empty State для Events:** Большой плейсхолдер с неброской иконкой календаря и текстом "You have no upcoming events. / У вас нет предстоящих событий." и кнопкой "Create Event". Обязательно в Qt использовать `QLabel` с выравниванием по центру и WordWrap.
- **Onboarding:** Окно `QDialog` в стиле мастера (`QWizard` или кастомный StackedWidget). Обязательно включает страницу "Выбор языка" самым первым шагом.

## 12. Accessibility and Keyboard Navigation
- Все `QPushButton` и `QLineEdit` должны иметь корректный `tabOrder`.
- В Strict режиме фокус автоматически устанавливается на кнопку "Snooze" (Отложить), а не на "Skip".
- Поддержка закрытия диалогов и нестрогих напоминаний клавишей `Esc`. (В Strict режиме `Esc` должен быть заблокирован).

## 13. High DPI and Resizing Notes
- **В Qt 6 масштабирование High DPI включено по умолчанию.** Однако, растровые форматы (.png) будут мылить.
- **Решение:** Использовать ТОЛЬКО векторные иконки (SVG). В Qt это отлично работает через `QIcon(":/icons/icon.svg")`.
- Окна не должны иметь жестко заданных размеров (`setFixedSize`), кроме маленьких диалогов. Вместо этого использовать `setMinimumSize` и `QVBoxLayout/QHBoxLayout` с `QSizePolicy::Expanding`.

## 14. Component Guidelines
- **Buttons:**
  - Primary (Сплошная заливка акцентным цветом). Используется для "Сохранить", "Создать".
  - Secondary (Обводка или легкий фон). "Отмена", "Отложить".
  - Destructive (Красный цвет). "Удалить правило", "Skip (в строгом режиме)".
- **Inputs:** `QSpinBox` для ввода минут (предотвращает ввод текста и ошибок парсинга), `QTimeEdit` для точного времени.
- **Spacing:** Использовать стандартные системные отступы Qt, либо задать через `QSS` (например, margin 16px, spacing 8px).

## 15. Visual Direction
- **Стиль:** Clean/Native. Избегать создания UI, похожего на "инопланетный корабль" из 2000-х.
- **Qt Реализация:** Использовать стиль **"Fusion"** (встроен в Qt) как базу для всех платформ (он выглядит современно и консистентно на Windows и Linux). Настроить `QPalette` для применения Светлой/Темной темы.
- **Шрифты:** Использовать системные шрифты (`QFontDatabase::systemFont(QFontDatabase::GeneralFont)`). Segoe UI на Windows, San Francisco/Inter/Ubuntu/Cantarell на Linux.

## 16. Open Questions / Risks
1. **Wayland (CachyOS / Arch Linux) vs X11:**
   - *Риск:* В Wayland протоколы безопасности запрещают приложениям принудительно перехватывать фокус, позиционировать окна поверх других и "блокировать" экран (без специальных shell-компонентов). Strict Mode (Полноэкранный оверлей) может не сработать должным образом в GNOME Wayland или KDE Wayland.
   - *Решение:* Для Wayland использовать максимально большое окно (`showMaximized()`) и запрашивать `QApplication::alert()` для привлечения внимания.
2. **Translation Overflow:**
   - *Риск:* Изменение языка "на лету" может сплющить UI, если компоненты не имеют гибкого Layout.
   - *Решение:* Использовать `QSpacerItem` и `WordWrap` на всех длинных `QLabel`.
3. **Background Process on Windows 10:**
   - Приложение должно потреблять минимум CPU (использовать `QTimer`, избегать `while(true)` лупов для проверки времени). Обязательно тестировать потребление RAM (в пределах 30-50 MB для Qt Core+GUI).
